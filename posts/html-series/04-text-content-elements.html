📚 Text & Content Elements

Headings, paragraphs, bold, italic, highlights, and more — the elements you'll use on literally every page you ever build.

<h1>–<h6><p><strong><em><mark><br> <hr>

The Content Layer

These are the elements that hold your words — your headings, your paragraphs, your emphasized points. They're the elements you'll reach for on every single page you build.

Headings: <h1> Through <h6>

HTML gives you six levels of headings. Headings create a content hierarchy that tells both browsers and screen readers how your information is organized.

<h1>Main Page TitleOne per page
<h2>Section HeadingMajor sections
<h3>SubsectionWithin an h2
<h4>Sub-subsectionLess common
<h5>Deep nestingRare
<h6>Deepest levelVery rare
🧠 Mindset Shift

Headings are not about making text bigger. That's CSS's job. Headings are about meaning and hierarchy. Use only one <h1> per page — it should describe the main topic.

Paragraphs: <p>

The <p> element is the most common element in HTML. Every block of body text goes in a paragraph tag.

HTML
<p>First paragraph.</p> <p>Second paragraph — automatically spaced.</p>

Text Formatting Elements

<strong> — Important Text

<strong> marks text as strongly important. Browsers render it bold, but the point is the meaning — screen readers change their tone, search engines give it extra weight.

HTML
<p>Do <strong>not</strong> skip the fundamentals.</p>
Browser Renders As
Do not skip the fundamentals.

<em> — Emphasized Text

<em> marks text with emphasis — a stress that changes the meaning of the sentence. Displays italic by default.

HTML
<p>You <em>can</em> do this. I promise.</p>
Browser Renders As
You can do this. I promise.

<mark> — Highlighted Text

<mark> highlights text like a yellow highlighter — draws attention to key terms.

HTML
<p>The most important rule: <mark>structure first, style later</mark>.</p>
Browser Renders As
The most important rule: structure first, style later.

<sup> and <sub>

HTML
<p>E = mc<sup>2</sup></p> <p>Water is H<sub>2</sub>O</p>
Browser Renders As
E = mc2

Water is H2O

Line Breaks & Horizontal Rules

<br> forces a line break within the same block. <hr> creates a thematic break — a visible horizontal line that separates sections.

💡 Pro Tip

Never use multiple <br> tags to create space. That's CSS's job. Use <br> only when content genuinely needs separate lines within the same block — like an address.

🔥 Dev_Fession

For an embarrassingly long time, I used <b> instead of <strong> and <i> instead of <em>. They look the same in the browser, so I figured they were the same thing.

They're not. <b> and <i> are purely visual. <strong> and <em> carry meaning. A screen reader will stress <em> text and announce <strong> text with urgency. Using the right tags is about caring enough to write code that works for everyone.

🏆 Mini Build: Styled Blog Post Intro

Create a blog intro using every text element from this lesson:

1
An <h1> post title and an <h2> subheading.
2
Two <p> paragraphs with at least one <strong> and one <em> in each.
3
A sentence using <mark> to highlight a key takeaway.
4
A <small> copyright line and an <hr /> separating sections.
5
Bonus: Use <sup> somewhere — a footnote, an ordinal, or an exponent.

Recap

Headings (<h1>–<h6>) create hierarchy. <p> holds body text. <strong> marks importance. <em> adds emphasis. <mark> highlights. <br> breaks a line, <hr> breaks a section. Every element is about meaning, not just appearance.
