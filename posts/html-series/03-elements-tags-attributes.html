The Three Words That Explain All of HTML

Every single piece of HTML is built from just three concepts: elements, tags, and attributes. Once you understand how these work together, you can read any HTML document on the internet.

🧠 Mindset Shift

Don't memorize every HTML tag — understand the pattern. Once you get how tags, elements, and attributes work together, learning any new tag is just filling in a template you already know.

Tags: The Labels

A tag is a keyword wrapped in angle brackets. Most tags come in pairs:

<p>This is a paragraph.</p>
Opening tagContentClosing tag
HTML
<h1>This is a heading</h1> <p>This is a paragraph</p> <strong>This is bold text</strong> <a href="https://codeherway.com">This is a link</a>
💡 Pro Tip

The most common beginner bug: forgetting the closing tag. If your bold text never stops being bold, check for a missing </strong>. The browser will try to "fix" it, usually in the worst possible way.

Elements: The Full Package

An element is the entire unit — opening tag, content, and closing tag together.

<p>Hello, world.</p>
This entire thing is one "element"
ℹ️ Good to Know

Elements can be nested inside each other. Nesting is how you build complex structures from simple pieces — HTML's superpower.

HTML
<p>I am learning <strong>HTML</strong> and it's <em>actually making sense</em>.</p>

Self-Closing Elements

Some elements are self-closing — they don't wrap around content. They are the content.

HTML
<img src="photo.jpg" alt="A sunset over the lake" /> <br /> <input type="text" name="username" /> <hr />

Attributes: The Details

Attributes go inside the opening tag using the pattern: name="value". Never in the closing tag.

class vs. id

class is reusable — same class on many elements. id must be unique — only one element per page. When in doubt, use a class.

HTML
<!-- class: reusable --> <p class="intro-text">First paragraph.</p> <p class="intro-text">Second paragraph, same style.</p> <!-- id: unique --> <section id="about"><h2>About Me</h2></section>
🔥 Dev_Fession

When I first started writing HTML, I treated attributes like decorations — I'd throw in a class when a tutorial told me to, but had no idea why. I'd leave alt empty on every image because "who reads that?"

Turns out: screen readers read that. Google reads that. Every empty alt="" I shipped was a silent "I don't care about the people using my website." I didn't mean it that way. But intent doesn't override impact.

Now I write alt text on every single image and actually read what attributes do before using them. It takes five extra seconds and makes your code accessible and professional.

Recap

Tags are the angle-bracket labels. Elements are the full package. Attributes are extra details in the opening tag using name="value" syntax. Together, these three concepts are the entire grammar of HTML.
