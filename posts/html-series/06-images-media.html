🖼️ Images

The <img> Element

Images are added to HTML with the <img /> element — a self-closing element because the image is the content. At minimum, it needs two attributes: src and alt.

HTML
<img src="photo.jpg" alt="Laptop on a desk with code on the screen" />

Why alt Text Is Non-Negotiable

The alt attribute provides alternative text that serves three critical purposes: accessibility (screen readers), fallback display when images fail to load, and SEO (search engines read alt text to understand images).

🖼️
Laptop on a desk with code on the screen
↑ This is what users see when an image can't load — your alt text fills the gap.
🧠 Mindset Shift

Writing alt text isn't a chore — it's a design decision. Shipping an image without alt text is like building a ramp and stopping two feet short of the door.

Writing Good alt Text

❌ Bad Alt Text

alt=""
Empty. Screen readers get nothing.
alt="image"
Useless — what image?

✅ Good Alt Text

alt="Woman coding on a laptop in a coffee shop"
Descriptive and specific.
alt="Bar chart showing 40% increase in signups"
Conveys the data the image communicates.
💡 Pro Tip

Ask yourself: "If I couldn't see this image, what would I need to know?" For decorative images that add no meaning, use alt="" — that tells screen readers to skip it.

Image Paths

The src attribute works exactly like href on links — it can be absolute (full URL) or relative (relative to your current file).

HTML
<!-- Absolute: external URL --> <img src="https://images.unsplash.com/photo.jpg" alt="Mountain at sunset" /> <!-- Relative: image in subfolder --> <img src="images/project-preview.webp" alt="Weather dashboard screenshot" /> <!-- Relative: go up one folder --> <img src="../images/logo.svg" alt="CodeHerWay logo" />

Image Formats: Which One to Use

.jpg
JPEG
Best for photos with lots of colors and gradients.
Photos, hero images
.png
PNG
Supports transparency. Great for logos and crisp edges.
Logos, screenshots, transparent graphics
.svg
SVG
Vector — scales to any size without quality loss.
Icons, logos, illustrations
.webp
WebP
Modern format. Smaller files than JPG/PNG at equal quality.
Everything — the modern default
💡 Pro Tip

WebP is your default format. 25–35% smaller than JPEG at the same quality, supports transparency. Use Squoosh (squoosh.app) to convert and compress before adding to your project.

Width & Height Attributes

Including width and height lets the browser reserve space for images before they load, preventing layout shift. Values are in pixels — no "px" unit needed.

HTML
<img src="project-screenshot.webp" alt="E-commerce product page screenshot" width="800" height="450" />
ℹ️ Good to Know

Always set width and height to the image's actual dimensions. This prevents Cumulative Layout Shift (CLS) — a Google ranking metric that measures how much a page jumps around while loading.

🔥 Dev_Fession

My first portfolio had a 4.2 MB hero image. Raw, uncompressed JPEG from Unsplash. No width or height. No alt text. The page took 8 seconds on mobile and the layout jumped around the whole time.

I ran Lighthouse. Performance score: 23 out of 100. I spent 15 minutes converting to WebP, compressing on Squoosh, adding width/height, and writing proper alt text. Score jumped to 89. Same image. Same page. Just proper HTML.

Performance is a feature. And the <img> tag is where most performance problems start.

Recap

<img /> needs src and alt — both required, both matter. Alt text serves accessibility, fallback display, and SEO. Default to WebP, use SVG for icons, PNG for transparency. Always add width and height. Compress everything before shipping.
