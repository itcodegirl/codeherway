🔗 HTML Links & Navigation

The <a> tag is what makes the web a "web." Without links, every page is an island. Let's learn how to connect everything together.

<a>hreftargetanchor linksmailto: / tel:

The Tag That Makes the Web a Web

The letter "H" in HTML stands for Hypertext — text that links to other text. The <a> tag (anchor tag) is how that comes to life. Every link you've ever clicked on any website was created with an <a> tag.

HTML
<a href="https://codeherway.com">Visit CodeHerWay</a>

The href attribute (short for hypertext reference) tells the browser where the link goes. Simple, powerful, used on every website.

Absolute vs. Relative Paths

Absolute Path

The full URL including https://. Use for external sites.

href="https://github.com/itcodegirl"

Relative Path

A path relative to where the current file lives. Use for your own pages.

href="about.html"
href="pages/contact.html"
href="../index.html"
HTML
<!-- Absolute: full URL to an external site --> <a href="https://github.com/itcodegirl">My GitHub</a> <!-- Relative: file in the same folder --> <a href="about.html">About Me</a> <!-- Relative: go up one folder --> <a href="../index.html">Back to Home</a>

Opening Links in a New Tab

HTML
<a href="https://github.com/itcodegirl" target="_blank" rel="noopener noreferrer"> My GitHub Profile </a>
ℹ️ Security Note

Always pair target="_blank" with rel="noopener noreferrer". Without it, the new tab has a reference back to your page that creates a minor security vulnerability. One extra attribute — always include it.

Anchor Links: Jumping Within a Page

Anchor links let you link to a specific section on the same page. Point the href to an element's id with a # symbol.

HTML
<nav> <a href="#about">About</a> <a href="#projects">Projects</a> <a href="#contact">Contact</a> </nav> <section id="about"><h2>About Me</h2></section> <section id="projects"><h2>My Projects</h2></section> <section id="contact"><h2>Contact</h2></section>
🧠 Mindset Shift

Anchor links are one of the first things that make a project feel like a real website. Clicking a nav item that scrolls you to a section — that's a legitimate UX pattern. And it's just an href pointing to an id. No JavaScript required.

Email & Phone Links

HTML
<!-- Opens email client --> <a href="mailto:hello@codeherway.com">Email Me</a> <!-- Pre-fill subject and body --> <a href="mailto:hello@codeherway.com?subject=Hello&body=I love your site!">Send a Message</a> <!-- Opens phone dialer on mobile --> <a href="tel:+13125551234">Call Us</a>
🔥 Dev_Fession

My first portfolio had a beautiful contact section with my email address displayed as plain text. Not a link — just text. You couldn't click it. You had to manually select it, copy it, open your email client, paste it, and send. I genuinely thought that was fine.

You know how many people emailed me through that contact section? Zero. The moment I turned it into a mailto: link, I actually started getting messages. Every point of friction you create is a person who won't follow through.

🏆 Mini Build: Portfolio Navigation

Build a single-page portfolio structure with working navigation:

1
A <nav> with anchor links to #about, #projects, and #contact.
2
Three <section> blocks with matching id attributes so the links actually work.
3
An external link to your GitHub with target="_blank" and rel="noopener noreferrer".
4
A mailto: link in your contact section.
5
A tel: link for a phone number.

Recap

The <a> tag connects everything on the web. Use absolute paths for external sites and relative paths for your own pages. Always include rel="noopener noreferrer" with target="_blank". Use anchor links (#id) for same-page navigation. Use mailto: and tel: for contact links. Every link is a UX decision — make them intentional.
