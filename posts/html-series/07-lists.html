📦 HTML Lists

Navigation menus, step-by-step instructions, feature lists, glossaries — lists are everywhere on the web. HTML gives you three types, and each one means something different.

<ul><ol><li><dl><dt> & <dd>

Three Types of Lists

HTML has three distinct list types, and each carries semantic meaning. Choosing the right list communicates the relationship between your items.

●
Unordered
<ul> + <li>
Items with no specific sequence. Order doesn't matter. Bullet points by default.
1.
Ordered
<ol> + <li>
Items in a meaningful sequence. Steps, rankings, chronology. Numbered by default.
→
Definition
<dl> + <dt> + <dd>
Term-and-description pairs. Glossaries, FAQs, metadata. The underused powerhouse.

Unordered Lists: <ul>

An unordered list is for items where order doesn't matter — grocery lists, skills, feature lists, navigation menus. Browsers render bullet points by default.

ℹ️ Good to Know

Navigation menus are almost always <ul> lists inside a <nav>. CSS transforms the bullets into a horizontal bar — but semantically, a nav menu is a list of links.

HTML
<nav> <ul> <li><a href="#about">About</a></li> <li><a href="#projects">Projects</a></li> <li><a href="#contact">Contact</a></li> </ul> </nav>

Ordered Lists: <ol>

An ordered list is for items where sequence matters — step-by-step instructions, rankings, recipes. Rearranging items would change the meaning.

💡 Pro Tip

start="5" changes the starting number. reversed counts down. type="A" switches to letters. Never hard-code numbers into <li> text.

Nested Lists

Lists can be nested by placing a new <ul> or <ol> inside an <li> — not after it.

HTML
<ul> <li>Frontend <ul> <li>HTML</li> <li>CSS</li> <li>JavaScript</li> </ul> </li> <li>Backend <ul> <li>Node.js</li> </ul> </li> </ul>
🧠 Mindset Shift

The nested <ul> goes inside the <li>, not after it. The parent <li> doesn't close until after the sub-list. If your nesting looks wrong, check that the inner list is actually inside the <li>.

Definition Lists: <dl>

The definition list pairs terms (<dt>) with their descriptions (<dd>). Perfect for glossaries, FAQs, product specs, and metadata.

Which List Type Should I Use?

Does the order of items matter?
Yes → <ol>
A collection with no sequence?
Yes → <ul>
Pairing terms with descriptions?
Yes → <dl>
Navigation menu?
<ul> inside <nav>
Steps in a tutorial or recipe?
Always <ol>
FAQ or glossary?
<dl> is the semantic choice
🔥 Dev_Fession

I used to build navigation menus with <div> tags — just a row of <a> links in a <div>. It looked fine. I did it that way for months.

Turns out screen readers announce "navigation, list, five items" for a proper <ul> in a <nav>. My <div>-based nav got read with no context, no count, no structure. I refactored every nav I'd ever built that week. It should have been done from the start.

🏆 Mini Build: Recipe Page

Build a recipe page using all three list types:

1
An <h1> recipe name and <p> intro.
2
An unordered list (<ul>) for the ingredients — order doesn't matter.
3
An ordered list (<ol>) for step-by-step instructions — order absolutely matters.
4
A definition list (<dl>) for "Quick Facts" — Prep Time, Servings, Difficulty paired with their values.
5
A <nav> at the top with anchor links to each section.
6
Bonus: Nest a <ul> inside one ingredient <li> for sub-ingredients.

Recap

Unordered lists (<ul>) for items where order doesn't matter. Ordered lists (<ol>) for sequential items. Definition lists (<dl>) for term-description pairs. Lists can be nested. Nav menus use <ul> inside <nav>. Always choose based on meaning, not appearance.
