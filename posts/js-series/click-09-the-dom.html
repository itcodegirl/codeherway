JavaScript DOM Manipulation

This is the aha moment. JavaScript stops being invisible and starts changing what you see on screen.

⏱ 11 min read 🎯 Complete Beginners 🛠 Mini Build Included
💬

This Is the Moment It Gets Real

Everything you've learned so far — variables, functions, arrays, objects — has been pure JavaScript logic running in isolation. Today that changes. The DOM is the bridge between your JavaScript and your HTML page. This is where code stops being invisible and starts doing things you can see.

🌐

What Is the DOM?

⏸ Pause & Process

Here's why querySelector is almost always the right choice: it uses CSS selector syntax you already know. "#submit-btn", ".card", "form input[type='email']" — all valid. It's more flexible than getElementById or getElementsByClassName, and it only returns one thing, which forces you to be specific.

When a browser loads your HTML, it parses it into a tree of objects called the Document Object Model. Every tag becomes a node. JavaScript can read, change, create, or remove any of those nodes in real time — without the page reloading.

ℹ️ Good to Know

The DOM isn't part of JavaScript — it's a browser API that JavaScript has access to. The document object is your entry point to everything on the page. Think of it as a live, editable map of your HTML.

🔍

Selecting Elements

JavaScript — selecting.js
// querySelector — returns the FIRST match (uses CSS selector syntax) const heading = document.querySelector("h1") const btn = document.querySelector("#submit-btn") // by id const card = document.querySelector(".card") // by class // querySelectorAll — returns ALL matches as a NodeList // NodeList supports forEach() and for...of, but NOT map/filter/reduce // Use Array.from(nodeList) if you need full array methods const allItems = document.querySelectorAll("li") allItems.forEach(item => console.log(item.textContent)) // Convert to array for full array methods const itemsArray = Array.from(allItems) const texts = itemsArray.map(item => item.textContent) // getElementById — old school but still fast const title = document.getElementById("main-title")
✏️

Modifying the DOM

⏸ Pause & Process

innerHTML is powerful and dangerous in the same breath. It lets you inject HTML strings directly into the DOM — which is great for rendering templates. But if that string contains user input, you've just opened a door to XSS attacks. Rule of thumb: use textContent for user-supplied text, innerHTML only for your own trusted templates.

JavaScript — modifying.js
const heading = document.querySelector("h1") // Text — safe, no HTML injection risk heading.textContent = "Updated Heading!" // innerHTML — ⚠️ NEVER put user-typed content here (XSS risk) // Only use with your own trusted strings heading.innerHTML = "Hello <em>World</em>" // Manage classes — preferred over inline styles heading.classList.add("active") heading.classList.remove("hidden") heading.classList.toggle("dark-mode") // adds if missing, removes if present heading.classList.contains("active") // returns true or false // Inline style — use sparingly, classList is usually better heading.style.color = "hotpink" // camelCase for CSS properties heading.style.fontSize = "2rem" // NOT font-size
💡 NodeList Is Not an Array

querySelectorAll() returns a NodeList, not an array. You can loop it with for...of, but array methods like .map() and .filter() won't work directly. Convert it first: Array.from(document.querySelectorAll("li")). Need a refresher on array methods? Post 07: Arrays has you covered.

➕

Creating New Elements

JavaScript — creating.js
// createElement — creates a new element (not on the page yet) const newCard = document.createElement("div") newCard.classList.add("card") newCard.textContent = "I was created by JavaScript!" // append — add it to the page document.querySelector(".container").append(newCard) // innerHTML for generating lots of HTML at once (your own content only) const list = document.querySelector("#user-list") const users = ["Jenna", "Maya", "Sara"] list.innerHTML = users.map(u => `<li>${u}</li>`).join("") // remove — delete an element document.querySelector(".old-banner")?.remove() // ?. prevents error if not found
⚠️ innerHTML + User Input = XSS

If you put text a user typed into innerHTML, a malicious user could inject scripts that run as part of your page. Always use textContent for any content that came from user input. Reserve innerHTML for your own trusted HTML template strings.

💡 Pro Tip

Prefer classList over style for visual changes. Define styles in CSS, toggle class names in JavaScript. This keeps your styling in CSS where it belongs — JS just controls which classes are applied.

⚡ DEV_FESSION

The first time I changed a heading's text with JavaScript and watched the page update without refreshing, I literally said "wait" out loud to no one. I'd been learning HTML and CSS for weeks and everything felt static. That one textContent change was the moment the whole thing clicked. JavaScript controls what you actually see.

🏆MINI BUILD — Button Changes the Headline

Connect JavaScript to HTML and make a button change what's on screen.

Your steps:
  • 01 Add an <h1 id="title">Original Heading</h1> to your HTML
  • 02 Add a <button id="change-btn">Change Heading</button>
  • 03 In JS: select both elements with querySelector
  • 04 When the button is clicked, change the heading's textContent
  • 05 Also change the color using classList.add() — define the color class in CSS
  • 06 Bonus: use createElement and append() to add a new <p> to the page when the button is clicked
