JavaScript Events

Users click, type, scroll, and submit. Events are how JavaScript listens and responds to every one of those actions.

⏱ 10 min read 🎯 Complete Beginners 🛠 Mini Build Included

Your Page Should React to People

In Post 09 you learned to change the DOM directly. But in real life, those changes happen because a user did something — clicked a button, typed in a field, submitted a form. Events are the mechanism that connects user actions to your JavaScript code.

🧱

addEventListener: The Right Way

⏸ Pause & Process

The event object is the part most beginners ignore. It's the first argument passed to every event handler — usually called e or event. It contains everything about what just happened: which element was clicked (e.target), which key was pressed (e.key), where the mouse was (e.clientX/Y). You'll use e.target and e.preventDefault() constantly.

JavaScript — events.js
// Syntax: element.addEventListener("eventType", callbackFunction) const btn = document.querySelector("#my-btn") btn.addEventListener("click", () => { console.log("Button clicked!") }) // The event object has info about what happened btn.addEventListener("click", (event) => { console.log(event.type) // "click" console.log(event.target) // the element that was clicked console.log(event.target.id) // "my-btn" }) // once: true — listener fires only once then auto-removes itself btn.addEventListener("click", handleClick, { once: true }) // removeEventListener — always pass the exact same function reference function handleClick() { console.log("clicked") } btn.addEventListener("click", handleClick) btn.removeEventListener("click", handleClick) // removes the listener
⚠️ Memory Leaks from Forgetting removeEventListener

Every listener you add stays active until explicitly removed or the element is deleted. If you add listeners inside a loop, or on elements that get replaced/re-created, you'll accumulate listeners and leak memory. Name your handler functions so you can pass the same reference to removeEventListener. Anonymous arrow functions can't be removed.

📋

Common Events You'll Use Daily

⏸ Pause & Process

Event delegation sounds advanced but the idea is simple: instead of attaching a listener to every child element, you attach one listener to the parent. When a child is clicked, the event bubbles up to the parent. You check e.target to know which child was clicked. This is how you handle clicks on elements that don't exist yet when the page loads.

click
User clicks an element
input
User types (fires every keystroke)
submit
Form is submitted
change
Value changes and loses focus
keydown
A key is pressed — check e.key
mouseover
Mouse hovers over element
DOMContentLoaded
HTML parsed; safe to query DOM
scroll
User scrolls the page
JavaScript — event-types.js
// Input — fires on every keystroke document.querySelector("#name-input").addEventListener("input", (e) => { console.log("Current value:", e.target.value) }) // Submit — always preventDefault to stop the page from refreshing document.querySelector("form").addEventListener("submit", (e) => { e.preventDefault() console.log("Form submitted without page refresh") }) // Keydown — use e.key for the key name (not deprecated keyCode) document.addEventListener("keydown", (e) => { if (e.key === "Escape") closeModal() if (e.key === "Enter") handleSubmit() })
🎯

Event Delegation: One Listener for Many Elements

Instead of adding a listener to every child element, add one listener to a parent and check which element was actually clicked via event.target. This is essential for dynamically-created elements — new DOM nodes don't have listeners on them unless you use delegation.

JavaScript — event-delegation.js
// Without delegation — breaks for dynamically added items document.querySelectorAll(".delete-btn").forEach(btn => { btn.addEventListener("click", handleDelete) // won't work on buttons added later }) // With delegation — one listener on the parent, works for all children document.querySelector("#todo-list").addEventListener("click", (e) => { if (e.target.matches(".delete-btn")) { e.target.closest("li")?.remove() } })
🧠 Callbacks — The Mental Model

Event listeners don't run immediately — they wait. When you write addEventListener("click", fn), you're saying "keep this function on standby and run it whenever a click happens." That waiting function is called a callback. JavaScript is full of callbacks — this is your first one.
