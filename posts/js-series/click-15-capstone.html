Project Blueprint

⏸ Pause & Process

If you get stuck, here's the debugging order: check the console, check the network tab, add console.log() at each step to see where the data stops flowing. Most bugs in projects like this fall into one of three categories: the element selector is wrong, the API response isn't shaped how you expect, or an async function is missing await. Check those three things first.

PHASE 01HTML Structure

Keep it minimal. Four elements are all you need: a title, a search input, a status message container, and a cards grid.

HTML — index.html
<body> <h1>User Dashboard</h1> <input type="text" id="search" placeholder="Search by name or city..."> <p id="status"></p> <div id="cards"></div> <button id="load-btn">Load Users</button> <script src="script.js"></script> </body>

ℹ️ About Debouncing

The search filter runs on every keystroke. For a local array of 10 users this is fine. If you were sending a real API request on each keypress, you'd debounce it — delay the request until the user stops typing for ~300ms. Worth knowing the pattern exists even if you don't need it here.
