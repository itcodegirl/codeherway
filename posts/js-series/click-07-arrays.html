Your Data Comes in Lists

Most real data isn't a single value. It's a list — a list of users, products, messages, todos, search results. JavaScript uses arrays to store ordered lists of values. Mastering arrays is what separates someone who can write basic JS from someone who can actually build things.

🧱

Creating & Accessing Arrays

⏸ Pause & Process

There's a distinction worth drilling in: mutating vs returning new. Methods like push(), pop(), and splice() change the original array. Methods like .map(), .filter(), and .slice() return a brand new array and leave the original alone. This matters the moment two parts of your code share the same array.

JavaScript — arrays.js
const colors = ["pink", "purple", "teal"] // Access by index — starts at 0, not 1 console.log(colors[0]) // "pink" console.log(colors[2]) // "teal" console.log(colors.length) // 3 console.log(colors[colors.length - 1]) // "teal" — last item trick // Add and remove (these MUTATE the original array) colors.push("gold") // adds to END → ["pink","purple","teal","gold"] colors.pop() // removes END → ["pink","purple","teal"] colors.unshift("red") // adds to START → ["red","pink","purple","teal"] colors.shift() // removes START → ["pink","purple","teal"]
⚠️ Mutating vs Non-Mutating Methods

push, pop, shift, unshift, splice, sort, and reverse all modify the original array. Most developers are caught off guard that sort() mutates — if you need a sorted copy, use [...arr].sort() instead.

⚡

Array Methods: The Real Power

JavaScript — array-methods.js
const nums = [1, 2, 3, 4, 5] // forEach — run a function on each item. Returns undefined. nums.forEach(n => console.log(n * 2)) // map — transform each item, returns a NEW array const doubled = nums.map(n => n * 2) console.log(doubled) // [2, 4, 6, 8, 10] — original unchanged // filter — keep only items that pass a test, returns a NEW array const evens = nums.filter(n => n % 2 === 0) console.log(evens) // [2, 4] // find — returns the FIRST item that matches (a single value, not an array) // returns undefined if nothing matches const firstBig = nums.find(n => n > 3) console.log(firstBig) // 4 // findIndex — like find, but returns the INDEX (or -1 if not found) const idx = nums.findIndex(n => n > 3) console.log(idx) // 3 (index of 4) // reduce — combine all items into a single value const total = nums.reduce((sum, n) => sum + n, 0) console.log(total) // 15 // includes — true/false: is this value in the array? console.log(nums.includes(3)) // true // some — true if ANY item passes the test console.log(nums.some(n => n > 4)) // true // every — true only if ALL items pass the test console.log(nums.every(n => n > 0)) // true
🧠 Mindset Shift

map and filter return new arrays. find and findIndex return a single value (or undefined/-1). None of them modify the original — that's immutability and it's a core modern JS pattern. push, pop, sort, and splice DO modify the original — that's the key difference.

⚡ DEV_FESSION

I wrote a for loop with an if statement inside every single time I needed to filter an array for probably four months. It worked, but I never understood why senior devs' code was so much shorter than mine. Then I discovered .filter(). A five-line block became a one-liner. Learn these methods early.
