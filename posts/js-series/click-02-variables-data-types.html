JavaScript Variables & Data Types

JavaScript needs to remember things. Variables are how you make that happen — and what you store in them matters.

⏱ 10 min read 🎯 Complete Beginners 🛠 Mini Build Included
💬

JavaScript Has a Memory Problem (On Purpose)

JavaScript is smart, but it doesn't remember anything on its own. Every time you tell it something — a name, a number, a true/false value — it forgets it the second it's done using it. Unless you store it somewhere.

That's what variables are for. A variable is a named container that holds a value so your code can use it later, change it, and pass it around.

🧱

let, const, and the One We Don't Talk About

⏸ Pause & Process

Stop here for a second. const vs let is one of those things that clicks immediately for some people and never quite clicks for others. The rule is simple: start with const. If you try to change it and JavaScript complains, switch to let. That's genuinely the whole decision process.

const
use by default
Value won't be reassigned. Use this for most things.
let
use when it changes
Value will be reassigned. Use for counters, user input.
var
don't use
Old way. Causes confusing scope bugs. You'll see it in legacy code.
JavaScript — variables.js
// const — for values that won't be reassigned const name = "Jenna" const maxScore = 100 // let — for values that will change let score = 0 let isLoggedIn = false // Reassigning let — totally fine score = 10 isLoggedIn = true // Reassigning const — throws an error ❌ // name = "someone else" ← TypeError: Assignment to constant variable
💡 Pro Tip

Default to const. Only reach for let when you know the value will change. This forces you to think about what should and shouldn't change — and that's a great habit.

Data Types: What Can You Store?

⏸ Pause & Process

typeof null returning "object" is a 30-year-old JavaScript bug that was never fixed because too much code depends on the broken behavior. You're not confused — it's just wrong. Remember it, laugh about it, move on.

String
"Hello" / 'World' / `Hi`
Text. Always wrapped in quotes.
Number
42 / 3.14 / -7
Any number — integers and decimals both.
Boolean
true / false
Only two values. No quotes. Used for decisions.
Undefined
let x
Declared but never given a value.
Null
let x = null
Intentionally empty — e.g. before an API call returns.
NaN
parseInt("hello")
Not a Number. Returned when math fails.
JavaScript — data-types.js
const username = "codegirlJenna" // String const age = 27 // Number const isPro = true // Boolean let nickname // Undefined (no value yet) let profilePic = null // Null (hasn't loaded yet) // typeof tells you what type a variable holds console.log(typeof username) // "string" console.log(typeof age) // "number" console.log(typeof isPro) // "boolean" console.log(typeof nickname) // "undefined" // NaN — what you get when math goes wrong const bad = parseInt("hello") console.log(bad) // NaN console.log(isNaN(bad)) // true console.log(typeof bad) // "number" ← famous JS quirk: NaN is typeof "number"
⚠️ Two Famous JS Quirks — Know These

typeof null === "object" — even though null means "no value," typeof null returns "object". A 30-year-old JavaScript bug.

typeof NaN === "number" — NaN stands for Not a Number, but its type is "number". Use isNaN(value) to check for it.

✏️

Naming Your Variables

JavaScript — naming.js
// ✅ Good naming — camelCase, descriptive const firstName = "Jenna" const userAge = 27 let isSubscribed = false // ❌ Bad naming — vague, confusing, or illegal const x = "Jenna" // what is x? // const 1stName = "Jenna" // can't start with a number ← SyntaxError // const my-name = "Jenna" // hyphens aren't allowed // const let = "Jenna" // reserved words can't be variable names
🧠 Mindset Shift

Good variable names are the first form of documentation. isLoggedIn tells you everything. x tells you nothing. Name things like you're writing notes for your future self — because you are.

⚡ DEV_FESSION

I named every variable x, y, and data for the first three months. Then I tried reading something I wrote two weeks earlier and had absolutely no idea what let d2 = x + y was doing. Descriptive names feel slow to type and immediately feel fast to understand.
