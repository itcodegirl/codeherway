JavaScript Operators & Comparisons

JavaScript can do math, compare values, and make logical decisions. These are the tools that make that happen.

⏱ 9 min read 🎯 Complete Beginners 🛠 Mini Build Included
💬

JavaScript Needs to Ask Questions

Your code constantly needs to ask things. Is this user logged in? Is the score above zero? Did the user type something in the box? To answer those questions, JavaScript uses operators — symbols that compare, combine, or transform values.

This post covers the three operator families you'll use constantly: arithmetic, comparison, and logical.

🧮

Arithmetic Operators

⏸ Pause & Process

Pause on === vs == for a moment. The rule is: always use ===. Triple equals checks both value AND type, so 5 === "5" is false. Double equals coerces types and produces results that confuse everyone, including seniors. Just use triple equals. Always.

JavaScript can do math. These work exactly like school — with one addition worth knowing: % (modulo), which gives you the remainder after division.

JavaScript — arithmetic.js
const a = 10 const b = 3 console.log(a + b) // 13 — addition console.log(a - b) // 7 — subtraction console.log(a * b) // 30 — multiplication console.log(a / b) // 3.33 — division console.log(a % b) // 1 — remainder of 10 ÷ 3 // Assignment shortcuts let score = 0 score += 10 // same as: score = score + 10 → 10 score -= 3 // same as: score = score - 3 → 7 score++ // adds 1 → 8 score-- // subtracts 1 → 7
⚠️ Type Coercion Gotcha

The + operator also joins strings. "5" + 3 gives "53", not 8 — because one operand is a string, JavaScript converts the number to a string too. Order doesn't matter either: 3 + "5" also gives "35". When mixing types, be deliberate. Use Number("5") + 3 if you need math.

⚖️

Comparison Operators

These compare two values and always return true or false. There's one critical rule: always use === instead of ==.

OperatorMeaningExampleResult
===Strict equal (value AND type)5 === 5true
!==Strict not equal5 !== 3true
==Loose equal — converts types first ⚠️5 == "5"true ← bug risk
>Greater than10 > 5true
<Less than3 < 1false
>=Greater than or equal5 >= 5true
<=Less than or equal4 <= 3false
🧠 === vs == — Always Strict

== converts types before comparing, so 5 == "5" is true even though one is a number and one is a string. That's a bug waiting to happen. === checks value AND type: 5 === "5" is false. Use ===. Always. No exceptions.

🔗

Logical Operators

⏸ Pause & Process

Truthy and falsy is one of those concepts that feels weird until it suddenly feels obvious. Six values are falsy: false, 0, "", null, undefined, and NaN. Everything else is truthy. Once you internalize this, reading conditional logic becomes effortless.

JavaScript — logical.js
const isLoggedIn = true const isPro = false const age = 22 // && (AND) — BOTH must be true console.log(isLoggedIn && isPro) // false — isPro is false console.log(isLoggedIn && age > 18) // true — both are true // || (OR) — at least ONE must be true console.log(isLoggedIn || isPro) // true — isLoggedIn is true console.log(false || false) // false — neither is true // ! (NOT) — flips the value console.log(!isLoggedIn) // false console.log(!isPro) // true
⚡

Truthy & Falsy: JavaScript's Hidden Rules

In JavaScript, every value is either truthy or falsy. Memorize the 6 falsy values. Everything else is truthy.

❌ Falsy values (only 6)
  • false
  • 0
  • "" (empty string)
  • null
  • undefined
  • NaN
✅ Truthy (everything else)
  • "hello" (any non-empty string)
  • 1, -1, 3.14 (any non-zero number)
  • [] (even an empty array)
  • {} (even an empty object)
  • true
JavaScript — truthy-in-practice.js
// Instead of this verbose check: if (username !== "" && username !== null && username !== undefined) { console.log("User exists") } // You can write this — same result, much cleaner: if (username) { console.log("User exists") } // ⚠️ Watch this gotcha — 0 is falsy! const cartCount = 0 if (cartCount) { // 0 is falsy — this NEVER runs console.log("Items in cart") } if (cartCount !== null) { // better: be specific about what you're checking console.log("Cart is defined") }
⚡ DEV_FESSION

I used == for probably six months before someone in a code review pointed out why my form validation kept letting empty strings through. Turns out "" == false is true in JavaScript. I had no idea. Switched to === everywhere and immediately fixed three bugs I didn't even know I had.
