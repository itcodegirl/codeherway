Stop Writing the Same Line Twelve Times

Loops let JavaScript repeat a block of code automatically — as many times as you need, without you writing it over and over. They're how you process lists, count things, generate repeated output, and handle data at scale.

🔁

The for Loop

⏸ Pause & Process

The for loop syntax has three parts separated by semicolons: start, condition, step. Read it out loud: 'start at 0; keep going while i is less than 5; add 1 each time.' Once you can narrate it, you can write any for loop. The syntax stops being cryptic.

JavaScript — for-loop.js
// for (start; condition; step) for (let i = 0; i < 5; i++) { console.log("Iteration:", i) // 0, 1, 2, 3, 4 } // Use break to stop early for (let i = 0; i < 10; i++) { if (i === 3) break console.log(i) // 0, 1, 2 — stops at 3 } // Use continue to skip an iteration for (let i = 0; i < 5; i++) { if (i === 2) continue // skip 2, but keep going console.log(i) // 0, 1, 3, 4 }
🔄

The while Loop

JavaScript — while-loop.js
let count = 0 while (count < 5) { console.log("Count:", count) count++ // CRITICAL: always update or you'll loop forever }
ℹ️ Infinite Loop Warning

An infinite loop is when the condition never becomes false — your browser tab will freeze. Always make sure something inside the loop moves it toward the exit condition. If your browser freezes, close the tab and fix your loop counter.

🗂️

for...of — The Modern Way to Loop Arrays

⏸ Pause & Process

Here's the practical guide to picking a loop: Use for...of when iterating arrays or NodeLists (modern, clean). Use for...in only for plain objects. Use a classic for loop when you need the index. Use while when you don't know how many iterations you'll need upfront. That covers 99% of real cases.

JavaScript — for-of.js
const languages = ["HTML", "CSS", "JavaScript"] // for...of — reads: "for each item in languages" for (const lang of languages) { console.log(lang) // "HTML", "CSS", "JavaScript" } // Also works with NodeLists from the DOM (Post 09) const items = document.querySelectorAll("li") for (const item of items) { console.log(item.textContent) }
🔑

for...in — For Objects (Not Arrays)

The for...in loop iterates over the keys of an object. It's designed for objects, not arrays — using it on arrays gives you the index numbers as strings, which is rarely what you want.

JavaScript — for-in.js
const user = { name: "Jenna", age: 27, city: "Chicago" } // for...in — iterates over the KEYS of an object for (const key in user) { console.log(key, user[key]) // "name" "Jenna" // "age" 27 // "city" "Chicago" } // ⚠️ Don't use for...in on arrays — use for...of instead const nums = [10, 20, 30] for (const i in nums) { console.log(i) // "0", "1", "2" ← keys as strings, not values } // You almost never want this. Use for...of for arrays.
🧠 Which Loop to Use

Use for...of for arrays. Use for...in for objects. Use for when you need the index number. Use while when you don't know how many times you'll loop.
