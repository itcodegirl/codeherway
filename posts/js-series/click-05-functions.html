The Most Important Concept in JavaScript

If you only understand one thing deeply from this entire series, make it functions. They are the foundation of all organized code. A function is a reusable block of logic you can name, call, and pass data to.

Without functions, you'd write the same code over and over. With them, you write it once and use it everywhere.

🧱

Three Ways to Write a Function

⏸ Pause & Process

Stop and appreciate what return does. It doesn't just end the function — it sends a value back to wherever the function was called from. That returned value can be stored, passed to another function, or used directly. Functions without return come back as undefined. This trips up beginners constantly.

JavaScript has three syntax forms for functions. They mostly do the same thing, but there are important differences you need to know — especially around hoisting.

JavaScript — functions.js
// 1. Function Declaration — HOISTED: can be called BEFORE it appears in the file function greet(name) { return `Hello, ${name}!` } console.log(greet("Jenna")) // "Hello, Jenna!" — works ✅ // 2. Function Expression — NOT hoisted: must be defined before you call it const sayHello = function(name) { return `Hello, ${name}!` } // sayHello("Jenna") called BEFORE this line would throw: ReferenceError // 3. Arrow Function — modern, concise, also NOT hoisted const hi = (name) => `Hello, ${name}!` // Single parameter? Parens are optional: name => `Hello, ${name}!` // Single expression? No curly braces needed — the result is returned implicitly // Arrow function — multi-line needs curly braces AND an explicit return const greetFull = (name) => { const msg = `Hello, ${name}!` return msg }
⚠️ Hoisting — The Key Difference

Only function declarations are hoisted (available before their definition line). Function expressions and arrow functions stored in const/let are NOT hoisted — calling them before their definition throws a ReferenceError. This is a real bug you will hit if you don't know this.

📨

Parameters, Arguments & Return Values

JavaScript — params-returns.js
// Parameters: name, age (placeholders in the definition) function introduce(name, age) { return `I'm ${name}, ${age} years old` } // Arguments: "Jenna", 27 (actual values passed in when calling) console.log(introduce("Jenna", 27)) // "I'm Jenna, 27 years old" // Default parameters — fallback if no argument is given function greetUser(name = "friend") { return `Hey ${name}!` } console.log(greetUser()) // "Hey friend!" console.log(greetUser("Jenna")) // "Hey Jenna!" // A function with no return — performs an action, returns undefined function logScore(score) { console.log("Score:", score) // no return → returns undefined } const result = logScore(10) console.log(result) // undefined — console.log is not the return value
🧠 return vs console.log

A function that returns a value sends something back so other code can use it. console.log just prints for you to see and returns undefined — the value vanishes. Always ask: do I need to use this result somewhere else? If yes, use return.

ℹ️ Rest Parameters vs arguments

Arrow functions don't have the old arguments object (you'll see this in older code). The modern way to accept any number of arguments is with the rest parameter: function sum(...nums) { return nums.reduce((a,b) => a+b, 0) }. The ... collects all arguments into a real array.
