JavaScript Objects Explained

Arrays have indexes. Objects have names. Learn to model real-world data with key-value pairs.

⏱ 10 min read 🎯 Complete Beginners 🛠 Mini Build Included
💬

Arrays Have Numbers. Objects Have Names.

Arrays are great for ordered lists. But real-world data usually has shape — a user has a name, an email, and an age. A product has a title, a price, and an in-stock status. That structured data needs named properties, not numbered indexes. That's what objects are for.

🧱

Creating & Accessing Objects

⏸ Pause & Process

The const rule trips people up with objects. const user = {} doesn't freeze the object — it just means you can't reassign user to point at a different object. You can freely add, change, or delete properties inside it. const protects the binding, not the contents.

JavaScript — objects.js
const user = { name: "Jenna", age: 27, isSubscribed: true, handle: "@itcodegirl" } // Dot notation — most common console.log(user.name) // "Jenna" // Bracket notation — use when the key is dynamic or stored in a variable const key = "handle" console.log(user[key]) // "@itcodegirl" // Add or update properties after creation // This works even with const — const prevents reassigning the variable, // NOT changing the object's properties. The object is still mutable. user.location = "Chicago" // adds a new property ✅ user.age = 28 // updates existing property ✅ // user = { name: "Maya" } ← this would throw — can't reassign const
ℹ️ Why const Objects Are Still Mutable

const prevents you from pointing the variable at a different object. It does not freeze the object itself. user.age = 28 is fine. user = {{ name: "Maya" }} throws a TypeError. This surprises almost every beginner.

🔑

Iterating Objects: Object.keys, values, entries

⏸ Pause & Process

Object destructuring is worth a full pause. const { name, age } = user does the same thing as const name = user.name; const age = user.age — just in one line. You'll use this constantly, especially when working with API responses and function parameters. Get comfortable with it now.

JavaScript — object-methods.js
const user = { name: "Jenna", age: 27, city: "Chicago" } // Object.keys() — array of property names console.log(Object.keys(user)) // ["name", "age", "city"] // Object.values() — array of property values console.log(Object.values(user)) // ["Jenna", 27, "Chicago"] // Object.entries() — array of [key, value] pairs console.log(Object.entries(user)) // [["name","Jenna"],["age",27],["city","Chicago"]] // Loop over entries — the most common pattern for (const [key, value] of Object.entries(user)) { console.log(`${key}: ${value}`) // "name: Jenna" // "age: 27" // "city: Chicago" }
🪆

Nested Objects & Methods

JavaScript — nested-objects.js
const user = { name: "Jenna", links: { github: "github.com/itcodegirl", site: "itcodegirl.com" }, // Method — a function inside an object // ⚠️ Use regular function syntax, NOT arrow function // Arrow functions don't bind 'this' — they'd lose access to the object introduce() { return `Hi, I'm ${this.name}!` } } console.log(user.links.github) // "github.com/itcodegirl" console.log(user.introduce()) // "Hi, I'm Jenna!" // The most common pattern — array of objects const products = [ { id: 1, name: "Hoodie", price: 49 }, { id: 2, name: "Sticker Pack", price: 9 }, ] console.log(products[0].name) // "Hoodie"
⚠️ Never Use Arrow Functions as Object Methods

Arrow functions don't have their own this — they inherit this from the surrounding scope, which inside an object literal is usually the global object or undefined. Always use regular function syntax (introduce() {{}} or introduce: function() {{}}) for object methods.

🧠 Array of Objects Is Everywhere

The pattern of an array of objects is how almost all real data is structured — API responses, database results, React state. Get comfortable with items[0].property and items.map(item => item.name). You'll write these hundreds of times.
