JavaScript if/else, Switch & Ternary

Code without decisions is just a list of instructions. Conditionals are where JavaScript starts thinking.

⏱ 10 min read 🎯 Complete Beginners 🛠 Mini Build Included
💬

Your Code Needs to Make Choices

Every interesting program makes decisions. Show this button only if the user is logged in. Display an error only if the field is empty. Change the greeting based on the time of day. This is control flow — and it starts with if/else.

🧱

if / else if / else

⏸ Pause & Process

Here's the thing about else if chains — JavaScript stops the moment it finds a true condition. It doesn't keep checking. That means order matters. Put your most specific checks first, your most general catch-all last. Flip them and you'll get logic bugs that are genuinely hard to spot.

JavaScript — conditionals.js
const age = 20 if (age >= 18) { console.log("You can vote") } else { console.log("Not old enough") } // Multiple conditions with else if const score = 82 if (score >= 90) { console.log("A") } else if (score >= 80) { console.log("B") // ← this one runs } else if (score >= 70) { console.log("C") } else { console.log("Try again") }
💡 Pro Tip

JavaScript stops checking conditions as soon as one is true. Once it finds a match, it runs that block and skips everything else. Order your conditions from most specific to least specific.

🔀

Switch Statements

⏸ Pause & Process

The ternary operator looks intimidating at first. Read it like a question: condition ? 'if true, use this' : 'if false, use this'. It's not trying to replace if/else — it's for assigning a value in one line. If you need to do multiple things, use if/else. If you just need one value, ternary is cleaner.

When you're checking one variable against many possible exact values, switch is cleaner than a chain of else ifs. Don't forget break — without it, JavaScript falls through to the next case and runs it too.

JavaScript — switch.js
const day = "Monday" switch (day) { case "Monday": console.log("Start of the week. You've got this.") break case "Friday": console.log("Almost there!") break case "Saturday": case "Sunday": console.log("Weekend mode") // handles both Saturday AND Sunday break default: console.log("Midweek grind") } // ⚠️ Missing break — fallthrough in action switch ("Monday") { case "Monday": console.log("Monday") // runs case "Tuesday": console.log("Tuesday") // ALSO runs — no break above break default: console.log("Other day") }
⚡

The Ternary Operator

The ternary is a one-line if/else. Perfect for simple true/false decisions, especially when assigning a value. Syntax: condition ? valueIfTrue : valueIfFalse

The ternary is an expression — it returns a value. You can store it in a variable or embed it in a template literal. You cannot put statements (like loops or variable declarations) inside a ternary.

JavaScript — ternary.js
const isLoggedIn = true // Long way — if/else let greeting if (isLoggedIn) { greeting = "Welcome back!" } else { greeting = "Please log in" } // Short way — ternary RETURNS a value (same result) const msg = isLoggedIn ? "Welcome back!" : "Please log in" console.log(msg) // "Welcome back!" // Great inside template literals const age = 20 console.log(`You are ${age >= 18 ? "an adult" : "a minor"}`) // Forward reference: the ?? operator (covered fully in Post 12) // is a cleaner alternative specifically for null/undefined checks const username = null const displayName = username ?? "Guest" // "Guest" — we'll cover ?? in Post 12
🧠 Mindset Shift

Use ternary for simple, single-value decisions. If you're doing multiple things in each branch, use a regular if/else — ternaries get unreadable fast when nested or complex.

⚡ DEV_FESSION

I didn't use the ternary operator for over a year because it looked weird and I didn't understand it. Then I saw someone's code with it and suddenly it clicked — it's just a compressed if/else that returns a value. Now I use it constantly for things like className={{isActive ? 'active' : ''}} in React.
