💬 "My Site Looks Fine But Also… Kind of Wrong?"

Your HTML is clean. Your layout isn't broken. But something still feels off — like your page looks like it was built by someone learning versus someone who knows what they're doing. You can't pinpoint exactly what it is.

Nine times out of ten? It's the typography.

Not the colors. Not the layout. The fonts. The line spacing. The size of the body text. The way the headings don't feel like headings. These details are invisible when they're right and unmistakable when they're wrong — and most beginners never get told which properties to touch.

Good typography makes beginner sites look intermediate. Instantly. Not because it's magic — it's because typography is what trained eyes notice first, and most beginners never get told to look at it. It's the difference between "someone made this" and "someone designed this."

💛 The Confidence Layer

Here's the thing nobody tells you: professional-looking typography isn't hard to achieve. It takes about eight properties, one Google Fonts import, and the willingness to actually look at your text. That's it. By the end of this post, your sites will look different — in a good way.

🧱 The Properties, One by One

1. font-family — Choosing Your Typeface

font-family sets which font is used. You can list multiple options separated by commas — this is called a font stack. The browser uses the first one it can find, falling through to the next if it can't load it. The last option should always be a generic family: serif, sans-serif, or monospace.

styles.css
CSS · font-family
body { /* Font stack: try each left to right */ font-family: 'Space Grotesk', system-ui, sans-serif; } h1, h2, h3 { /* Headings can use the same or a different face */ font-family: 'Space Grotesk', sans-serif; } code, pre { /* Code always uses a monospace font */ font-family: 'Fira Code', 'Courier New', monospace; } /* Without a Google Fonts import, custom fonts won't load. We'll cover that in the Google Fonts section below. */

2. font-size — Size with Intention

The browser default is 16px for body text. That's your baseline. Everything you set in CSS is relative to that — or it should be.

3. rem vs px — Which One to Use

px is an absolute unit — 16px is always 16px no matter what. rem is relative to the root font size (the html element). By default, 1rem = 16px.

styles.css
CSS · rem vs px
/* px — fixed, predictable, doesn't scale */ body { font-size: 16px; /* always 16px, ignores user preferences */ } /* rem — relative to root, scales with user settings */ html { font-size: 16px; /* set the root, 1rem = 16px */ } body { font-size: 1rem; /* = 16px */ } h2 { font-size: 1.75rem; /* = 28px — scales proportionally */ } small { font-size: 0.875rem;/* = 14px */ } /* Use rem for font-size. Use px for borders, shadows, fine details. */
✦ The Practical Rule

Use rem for all font sizes and most spacing. Use px for borders, box shadows, and other small precise details that shouldn't scale. This gives you fluid, accessible text sizing with precise control over decorative details.

4. font-weight — Light to Heavy

styles.css
CSS · font-weight
/* Named values */ .light { font-weight: 300; } /* thin, airy */ .regular { font-weight: 400; } /* body text */ .medium { font-weight: 500; } /* slightly heavier body */ .semibold { font-weight: 600; } /* labels, UI */ .bold { font-weight: 700; } /* headings */ .black { font-weight: 900; } /* display, impact */ /* The font must have these weights loaded — a font loaded with only 400 & 700 won't render 300 or 600 */

5. line-height — The Most Underrated Property

line-height controls the space between lines of text. It is, without question, the most impactful typography property that beginners ignore. Too tight and text feels claustrophobic. Too loose and it falls apart. The sweet spot for body text is usually 1.6 to 1.8.

styles.css
CSS · line-height
body { line-height: 1.75; /* unitless — relative to font-size. Best practice. */ } h1, h2, h3 { line-height: 1.15; /* headings need tighter spacing — they're large */ } .lead { line-height: 1.6; /* intro text, slightly open */ } /* Rule of thumb: Headings → 1.1 – 1.25 (tight, impactful) Body text → 1.6 – 1.8 (readable, airy) Captions → 1.4 – 1.5 (compact but clear) */

6. letter-spacing — Presence and Breath

letter-spacing controls the space between individual characters. Small adjustments make a massive difference in feel. Negative values tighten the text — great for bold display headlines. Positive values open it up — essential for small uppercase labels.

7. text-transform — Control the Case

styles.css
CSS · text-transform
.kicker { text-transform: uppercase; /* LOOKS LIKE THIS */ letter-spacing: 2px; /* always pair with letter-spacing */ font-size: 0.75rem; /* and a smaller size */ } .title-case { text-transform: capitalize; /* Capitalizes Every Word */ } .lowercase { text-transform: lowercase; /* forces all lowercase */ } /* text-transform is CSS-only — the HTML stays unchanged. You can write lowercase in the file and display UPPERCASE in the browser. */
✦ The Uppercase + Letter-spacing Combo

Whenever you use text-transform: uppercase, also add letter-spacing: 1.5px or more. Uppercase letters set too tightly look cramped — they need air. This combination is how every professional "eyebrow" label or category tag is styled. It's a small thing that reads as very intentional.

8. Google Fonts — Free Custom Typefaces in 3 Lines

Google Fonts gives you hundreds of free, high-quality typefaces. You import them with a <link> tag in your HTML head, then reference them by name in your CSS. Here's the complete workflow:

index.html
HTML · Step 1 — Import the font
<head> <!-- 1. Go to fonts.google.com 2. Pick a font, select the weights you need 3. Copy the link tag they give you, paste here --> <link rel="preconnect" href="https://fonts.googleapis.com"> <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin> <link href="https://fonts.googleapis.com/css2?family=Space+Grotesk:wght@400;600;700&display=swap" rel="stylesheet"> <link rel="stylesheet" href="styles.css"> </head>
styles.css
CSS · Step 2 — Use it by name
body { /* Use the exact name from Google Fonts, in quotes */ font-family: 'Space Grotesk', sans-serif; } /* That's it. The font loads from Google's CDN and applies to every element that inherits from body. */

How Browsers Use CSS

When a browser loads a page, it reads your HTML structure first, then applies CSS rules to decide how everything looks. The two languages are always working together — HTML says what it is, CSS says how it appears.

Before CSS, every webpage looked like a university document from 1996. After CSS, the same content can become anything. That's not decoration. That's control.

CSS Beginner Typography
✦ Notice What You Did

Three distinct text colors (white headline, muted lead, gray body). Two different line-heights. Uppercase + letter-spacing on the kicker. Negative letter-spacing on the headline. All of it working together to create visual hierarchy. That's what typography is — a system of decisions, not random font choices.

💛 Typography Is the Fastest Upgrade You'll Ever Make

People who look at a website and say "this looks professional" are almost always responding to typography before anything else — before color, before layout, before images. It's the first thing trained eyes see and the last thing beginners think to fix.

Now you're thinking about it. Here's what you now have that you didn't before:

  • font-family with a proper font stack — and Google Fonts loaded correctly
  • font-size in rem so text scales like it's supposed to
  • line-height set intentionally for both headings and body text
  • letter-spacing that makes uppercase labels and display text breathe
  • text-transform for consistent, CSS-driven case control
  • A type hierarchy system: kicker → headline → lead → body → meta
  • A finished blog article that looks like something you built on purpose

🧾 DevFession

My first five projects all used Arial. Not because I made a choice — because I never changed it. The browser decided. I just let it.

And the worst part is I didn't even know that's why my sites looked off. I'd spend hours tweaking colors and layouts thinking something was wrong with those. Colors were fine. Layout was fine. My typography was just completely unaddressed, and it was making everything else look amateur by association.

The day I figured out line-height: 1.75 changed my body text was the day I genuinely felt like I understood CSS for the first time. It was such a small thing. Such an absurdly small thing. But suddenly my text had air — it had room to breathe. And the whole page felt different.

If your site looks "almost right," go look at your typography first. Nine times out of ten, that's where the answer is hiding.
